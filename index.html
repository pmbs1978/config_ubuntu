Guia Completo: Configurando o Ambiente de Desenvolvimento Laravel no Ubuntu

Este guia interativo detalha o processo de configuração de um ambiente de desenvolvimento completo para projetos Laravel no Ubuntu, utilizando Docker, Sail, VS Code e mais.

Passo 1: Atualizar o Sistema

Primeiro, vamos garantir que todos os pacotes do seu sistema estejam atualizados. 

sudo apt update && sudo apt upgrade -y

Passo 2: Configurações Iniciais do Ubuntu

Estas são algumas configurações recomendadas para melhorar a usabilidade e segurança do seu sistema após uma nova instalação.

1. Ativar Drivers Proprietários

Para garantir o melhor desempenho de hardware (especialmente placas gráficas), abra a aplicação "Software & Updates" → "Additional Drivers", selecione o driver testado e aplique as alterações.

2. Ativar "Minimizar ao Clicar" na Dock

Permite minimizar uma janela ao clicar no seu ícone na dock. 

gsettings set org.gnome.shell.extensions.dash-to-dock click-action 'minimize'

3. Instalar o GNOME Tweaks (Ajustes)

Permite personalizar aspetos avançados do ambiente de trabalho. 

sudo apt install -y gnome-tweaks

4. Ativar o Firewall (UFW)

É uma boa prática de segurança ativar o firewall. Os comandos abaixo ativam e permitem tráfego web padrão. 

sudo ufw enable
sudo ufw status
sudo ufw allow http
sudo ufw allow https

Passo 3: Instalação de Ferramentas e Aplicações Essenciais

Instalação de um conjunto de ferramentas de linha de comando e aplicações gráficas úteis.

1. Ferramentas Essenciais de Linha de Comando 

sudo apt install -y curl wget git xz-utils

2. Navegador Google Chrome 

wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo apt install ./google-chrome-stable_current_amd64.deb -y
rm google-chrome-stable_current_amd64.deb

Extensões Recomendadas para o Chrome

Após instalar o Chrome, considere adicionar estas extensões para aumentar a sua produtividade:

  • Bitwarden: Um gestor de senhas de código aberto e seguro para guardar todas as suas credenciais.
  • GoFullPage: Captura de ecrã de página inteira com um único clique.
  • ChatHub: Um cliente "tudo-em-um" que permite usar vários chatbots (como ChatGPT, Gemini, etc.) na mesma interface.

3. Terminal Wave 

sudo snap install wave-terminal

4. Aplicações de Produtividade e Gráficos

Instala o editor de imagens GIMP e o cliente FTP FileZilla. 

sudo apt install -y gimp filezilla

Instalar o LibreOffice (via Snap)

Para garantir a versão mais recente e atualizações automáticas, é recomendado remover a versão pré-instalada (se existir) e instalar a versão Snap.

Primeiro, remova completamente a versão existente: 

sudo apt purge --autoremove libreoffice* -y

Depois, instale a versão Snap: 

sudo snap install libreoffice

5. Multimédia: VLC e Codecs

Instalar o VLC (via Snap)

Instalar o VLC através do Snap garante que tem a versão mais recente. 

sudo snap install vlc

Instalar Codecs Multimédia Adicionais

Nota: A instalação dos codecs restritos irá pedir que aceite os termos da licença (EULA). Use a tecla Tab para navegar e Enter para confirmar. 

sudo apt install -y ubuntu-restricted-extras

6. Ferramentas Python: pip e pipx 

sudo apt install -y python3-pip pipx
pipx ensurepath

7. VPN: Private Internet Access (PIA)

Aceda à página de downloads da PIA, descarregue o instalador (.run) e execute os comandos abaixo na sua pasta de Downloads. 

cd ~/Downloads
chmod +x pia-linux-*.run
./pia-linux-*.run

8. Instalar o Gemini CLI (Escolha uma opção)

Opção 1: Via Script (Recomendado para instalação geral)

Este método não requer Node.js e instala o CLI diretamente no sistema. 

curl -sS https://ai.google.dev/assets/cli/install.sh | bash
Opção 2: Via NPM (Para utilizadores de Node.js)

Se prefere gerir ferramentas através do NPM, use este comando. 

npm install -g @google/gemini-cli

Após a instalação (com qualquer um dos métodos), reinicie o terminal e configure a sua chave de API: 

gemini init

Passo 4: Fluxo de Trabalho Essencial com Git e GitHub

Depois de instalar o Git, este é o fluxo de trabalho básico para clonar, receber e enviar alterações para um repositório no GitHub.

1. Clonar um Repositório do GitHub

"Clonar" é o processo de criar uma cópia local exata de um repositório. Na página do repositório no GitHub, clique no botão verde `< > Code` e copie o URL HTTPS. 

git clone https://github.com/seu-usuario/seu-repositorio.git

2. O Fluxo de Trabalho de Sincronização

A. Receber Atualizações do GitHub (git pull)

Antes de começar a trabalhar, garanta que a sua cópia local está atualizada. 

cd nome-do-repositorio
git pull origin main

B. Enviar as Suas Alterações (add, commit, push)

Siga estes três passos para enviar as suas modificações.

Adicionar Ficheiros (indicar ao Git o que guardar): 

git add .

Registar as Alterações (criar um "ponto de controlo"): 

git commit -m "Mensagem descritiva, ex: Adiciona a funcionalidade X"

Enviar para o GitHub: 

git push origin main

3. Resolver Erros de Autenticação com PAT

O GitHub já não aceita senhas para operações no terminal. É necessário usar um Personal Access Token (PAT).

O Erro Comum:

remote: Invalid username or token. Password authentication is not supported for Git operations.
fatal: Authentication failed for 'https://github.com/...'

Como Criar e Usar um PAT:

  1. Faça login no GitHub, clique na sua foto de perfil → Settings.
  2. No menu esquerdo, vá a Developer settings → Personal access tokens → Tokens (classic).
  3. Clique em Generate new token (e depois em "Generate new token (classic)").
  4. Dê um nome, defina uma data de validade e, em Scopes, marque a caixa repo.
  5. Clique em Generate token e copie o token imediatamente para um local seguro.
  6. Volte ao terminal. Quando executar `git push` ou `git pull`, use:
    • Username: o seu nome de utilizador do GitHub.
    • Password: cole o Personal Access Token (PAT).

Como Guardar o Token no Linux

Para não ter de inserir o token sempre, pode guardá-lo em cache ou de forma permanente.

1. Guardar em cache por um tempo (Recomendado)

Guarda o token na memória por um período (ex: 1 hora). É mais seguro. 

git config --global credential.helper 'cache --timeout=3600'

Pode aumentar o 3600 para mais segundos (ex: 86400 para 24 horas).

2. Guardar permanentemente (Menos seguro)

Guarda o token num ficheiro de texto simples na sua pasta pessoal. É mais conveniente, mas menos seguro. 

git config --global credential.helper store

Passo 5: Melhorar o Terminal (Opcional)

Esta seção é opcional, mas altamente recomendada para uma experiência de terminal mais produtiva e visualmente agradável com Tilix, ZSH e Powerlevel10k.

1. Instalar o Terminal Tilix 

sudo apt install -y tilix

2. Instalar a Fonte Fira Code

Fira Code é uma fonte popular para programação que suporta "ligatures", que combinam múltiplos caracteres em um único símbolo (ex: `=>` torna-se `⇒`). É ótima para o terminal e para o VS Code. 

sudo apt install -y fonts-firacode

3. Instalar ZSH 

sudo apt install -y zsh

4. Instalar Oh My Zsh

Quando perguntado, confirme que deseja tornar o ZSH o seu shell padrão. 

sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

5. Instalar as Fontes MesloLGS NF

Estas fontes são essenciais para os ícones do Powerlevel10k. Depois de instalar, **configure a fonte do seu terminal (Tilix) para `MesloLGS NF Regular`**. 

mkdir -p ~/.local/share/fonts
cd ~/.local/share/fonts
curl -L -O https://github.com/romkatv/powerlevel10k-media/raw/master/MesloLGS%20NF%20Regular.ttf
curl -L -O https://github.com/romkatv/powerlevel10k-media/raw/master/MesloLGS%20NF%20Bold.ttf
curl -L -O https://github.com/romkatv/powerlevel10k-media/raw/master/MesloLGS%20NF%20Italic.ttf
curl -L -O https://github.com/romkatv/powerlevel10k-media/raw/master/MesloLGS%20NF%20Bold%20Italic.ttf
cd ~
fc-cache -f -v

6. Instalar o tema Powerlevel10k 

git clone --depth=1 https://github.com/romkatv/powerlevel10k.git ${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/themes/powerlevel10k

7. Instalar Plugins (Sugestões e Destaques) 

git clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions
git clone https://github.com/zsh-users/zsh-syntax-highlighting.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

8. Configurar `~/.zshrc`

Abra `~/.zshrc` e edite as seguintes linhas: 

ZSH_THEME="powerlevel10k/powerlevel10k"
plugins=(git zsh-autosuggestions zsh-syntax-highlighting)

Feche e reabra o terminal. O assistente de configuração `p10k configure` deve iniciar. Siga os passos para personalizar a aparência.

Passo 6: Instalar o PHP e Extensões

Instalação local do PHP e extensões comuns, úteis para executar o Composer e comandos Artisan iniciais. 

sudo apt install -y php-cli php-mbstring php-xml php-bcmath php-curl php-zip unzip
php -v

Passo 7: Instalar o Composer

Instalação do gestor de dependências do PHP. 

php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"
php -r "if (hash_file('sha384', 'composer-setup.php') === 'e21205b207c3ff031906575712edab6f13eb0b361f2085f1f1237b7126d785e826a450292b6cfd1d64d92e6563bbde02') { echo 'Instalador verificado'; } else { echo 'Instalador corrompido'; unlink('composer-setup.php'); } echo PHP_EOL;"
php composer-setup.php
php -r "unlink('composer-setup.php');"
sudo mv composer.phar /usr/local/bin/composer
composer --version

Passo 8: Instalar o Node.js e NPM (via NVM)

Usar o NVM (Node Version Manager) é a melhor abordagem para gerir múltiplas versões do Node.js. Após executar o primeiro comando, feche e reabra o terminal. 

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

Depois de reabrir o terminal, execute: 

nvm install --lts
node -v && npm -v

Passo 9: Instalar o Docker

Instalação do Docker Engine para gerir os contentores do Sail.

1. Configurar o repositório do Docker 

sudo apt-get install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

2. Instalar o Docker Engine 

sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

3. Adicionar o seu usuário ao grupo `docker`

Importante: Faça logout e login novamente no sistema para que a alteração de grupo tenha efeito. 

sudo usermod -aG docker $USER

Passo 10: Criar um Novo Projeto Laravel

Criação de um novo projeto Laravel usando o Composer. 

mkdir -p ~/seus-projetos
cd ~/seus-projetos/
composer create-project laravel/laravel meu-app-laravel

Passo 11: Configurar e Usar o Laravel Sail

O Sail é uma interface de linha de comando para interagir com o ambiente de desenvolvimento Docker padrão do Laravel. 

cd meu-app-laravel
composer require laravel/sail --dev
php artisan sail:install

Após selecionar os serviços (ex: mysql, redis), inicie os contentores: 

./vendor/bin/sail up -d

Passo 12: Instalar e Configurar o Visual Studio Code

Instalação do VS Code e configuração de extensões recomendadas para desenvolvimento Laravel.

1. Instalar o VS Code 

sudo snap install --classic code

2. Extensões Recomendadas

Abra a aba de Extensões (Ctrl+Shift+X) no VS Code e instale:

  • PHP Intelephense (`bmewburn.vscode-intelephense-client`)
  • Laravel Blade Snippets (`onecentlin.laravel-blade`)
  • Laravel Extra Intellisense (`amiralizadeh9480.laravel-extra-intellisense`)
  • Laravel Go To View (`codingyu.laravel-goto-view`)
  • Docker (`ms-azuretools.vscode-docker`)
  • Dev Containers (`ms-vscode-remote.remote-containers`)
  • GitHub Copilot (`GitHub.copilot`)
  • GitHub Pull Requests and Issues (`GitHub.vscode-pull-request-github`)
  • Material Icon Theme (`PKief.material-icon-theme`)
  • DotENV (`mikestead.dotenv`)
  • Prettier (`esbenp.prettier-vscode`)
  • Tailwind CSS IntelliSense (`bradlc.vscode-tailwindcss`)
  • Live Server (`ritwickdey.liveserver`)
  • CSS Formatter (`aeschli.vscode-css-formatter`)

3. Configurações (`settings.json`)

Pressione `Ctrl+Shift+P`, procure por `Open User Settings (JSON)` e adicione as seguintes configurações para otimizar o seu ambiente: 

{
    "workbench.startupEditor": "none",
    "files.autoSave": "onFocusChange",
    "editor.fontFamily": "'Fira Code', monospace",
    "editor.fontLigatures": true,
    "editor.fontSize": 16,
    "editor.formatOnSave": true,
    "editor.defaultFormatter": "esbenp.prettier-vscode",
    "[php]": {
        "editor.defaultFormatter": "bmewburn.vscode-intelephense-client"
    },
    "files.associations": {
        ".env.*": "dotenv",
        ".blade.php": "blade"
    },
    "terminal.integrated.fontFamily": "MesloLGS NF"
}
  • workbench.startupEditor: Impede que o ecrã de boas-vindas apareça ao iniciar o VS Code.
  • files.autoSave: Guarda os ficheiros automaticamente sempre que muda de janela ou de ficheiro.
  • editor.fontFamily & fontLigatures: Define a Fira Code como fonte padrão do editor e ativa as "ligatures" para uma melhor experiência de leitura de código.

Passo 13: Tutorial: Editar Ficheiros via FTP/SFTP no VS Code

Trabalhar diretamente em ficheiros num servidor remoto a partir do Visual Studio Code é uma das formas mais eficientes de desenvolver para a web.

Parte 1: Instalação da Extensão Correta

A extensão "SFTP" mais popular (do autor liximomo) foi descontinuada. Este tutorial utiliza a sua sucessora, publicada pelo autor Natizyskunk.

  1. Abra o VS Code e aceda à vista de Extensões (Ctrl+Shift+X).
  2. Na barra de pesquisa, procure por SFTP.
  3. Instale a extensão publicada por Natizyskunk.

Ação Necessária: Se já tinha a versão antiga da extensão (do autor liximomo), deve desinstalá-la para evitar conflitos.

Parte 2: Configuração da Ligação ao Servidor

  1. Certifique-se de que tem a pasta do seu projeto local aberta no VS Code.
  2. Abra a Paleta de Comandos com Ctrl+Shift+P.
  3. Escreva SFTP: Config e pressione Enter.
  4. Isto irá criar e abrir o ficheiro .vscode/sftp.json no seu projeto.

Parte 3: Preencher os Detalhes da Ligação em `sftp.json`

Este ficheiro JSON contém os dados da sua ligação. Use o exemplo apropriado abaixo.

Exemplo para uma ligação SFTP (Recomendado): 

{
    "name": "Meu Servidor SFTP",
    "host": "ssh.meusite.com",
    "protocol": "sftp",
    "port": 22,
    "username": "o_seu_utilizador_ssh",
    "password": "a_sua_password_aqui",
    "remotePath": "/var/www/meusite/",
    "uploadOnSave": true
}

Exemplo para uma ligação FTP (Menos Seguro): 

{
    "name": "Meu Servidor FTP",
    "host": "ftp.meusite.com",
    "protocol": "ftp",
    "port": 21,
    "username": "o_seu_utilizador_ftp",
    "password": "a_sua_password_aqui",
    "remotePath": "/public_html/",
    "uploadOnSave": true
}

Explicação das Configurações:

  • "name": Um nome à sua escolha para identificar a ligação.
  • "host": O endereço do seu servidor (IP ou domínio).
  • "protocol": Defina como "ftp" ou "sftp".
  • "port": A porta correta (normalmente 21 para FTP, 22 para SFTP).
  • "username": O seu nome de utilizador.
  • "password": A sua senha. (Para mais segurança, pode apagar esta linha e a extensão irá pedi-la sempre que se ligar).
  • "remotePath": A pasta de destino no servidor (ex: /public_html/).
  • "uploadOnSave": Se true, envia o ficheiro automaticamente sempre que o guarda.

Parte 4: O Fluxo de Trabalho Diário

  1. Na barra de atividade esquerda, clique no novo ícone do SFTP.
  2. Clique com o botão direito no nome do seu servidor e escolha Connect para ver os ficheiros remotos.
  3. Clique num ficheiro remoto para o abrir e editar.
  4. Quando guardar (Ctrl+S), o ficheiro será enviado automaticamente para o servidor.

Final: Comandos Comuns do Sail e Alias

Para simplificar o uso do Sail, crie um alias. Adicione o seguinte ao seu `~/.zshrc` (ou `~/.bashrc`): 

alias sail='[ -f sail ] && bash sail || bash vendor/bin/sail'

Recarregue o terminal (`source ~/.zshrc`) e agora pode usar `sail` em vez de `./vendor/bin/sail`.

  • Iniciar/Parar: `sail up -d` / `sail down`
  • Migrações: `sail artisan migrate`
  • Composer: `sail composer require ...`
  • NPM: `sail npm install && sail npm run dev`

O seu ambiente de desenvolvimento está configurado e pronto para uso!
